# How To Create PPT Using template.pdf

This guide explains exactly how to create your final submission PPT from the template.

## 1. Files You Need

Use these files from the project folder:

- `template.pdf` (design reference)
- `PPT_TEMPLATE_FILLED_CONTENT.md` (ready slide content)
- `assets/images/ps3-architecture.svg` (architecture visual)

## 2. Quick Workflow (30-45 Minutes)

1. Open PowerPoint (or Google Slides).
2. Create a new 16:9 presentation.
3. Recreate template style from `template.pdf`:

- same background style
- same title font style
- same accent colors

4. Create 12 slides.
5. Copy content slide-by-slide from `PPT_TEMPLATE_FILLED_CONTENT.md`.
6. Insert the architecture image in Slide 4.
7. Add your team name and members in Slide 1.
8. Export to PDF and PPTX.

## 3. Recommended Slide Structure

Use this exact sequence:

1. Title
2. Problem
3. Existing gaps
4. Solution overview
5. Event taxonomy
6. Scoring and explainability
7. Fairness and context
8. Hardware-software split
9. Data and validation plan
10. Metrics and targets
11. Risks and mitigation
12. Roadmap and ask

## 4. Visual Rules (Important)

- Max 5 bullets per slide.
- Keep each bullet 1 line where possible.
- Use one strong visual per technical slide.
- Avoid heavy paragraphs.
- Keep font size readable from distance.

## 5. Slide Design Checklist

Before final export, verify:

- Same font family and size pattern across slides
- Consistent margins and alignment
- Same icon/color style across all slides
- No overcrowded slide
- No spelling errors in headings

## 6. Technical Accuracy Checklist

Must be consistent with your docs:

- PS3 only (no PS2/PS6 references)
- Stage 1 = PPT idea submission
- Stage 2 = implementation only if selected
- Sensor feasibility depends on hardware team sign-off
- Score formula appears exactly once and consistently

## 7. Presenter Script Flow (4-5 Minutes)

Time split suggestion:

- Slides 1-3: 60 seconds
- Slides 4-7: 120 seconds
- Slides 8-11: 90 seconds
- Slide 12: 30 seconds

Closing sentence:
"Our concept is implementation-ready, measurable, and realistic, with hardware feasibility gated through explicit team sign-off."

## 8. Export and Submission

Final output files:

- `PS3_Submission_Final.pptx`
- `PS3_Submission_Final.pdf`

Before submission:

1. Run one full rehearsal.
2. Ask one teammate to cross-check technical consistency.
3. Ensure all images render correctly in PDF export.

## 9. If You Need a Faster Version

For strict time limit, reduce to 8 slides:

- Merge Slide 2 + 3
- Merge Slide 5 + 6
- Merge Slide 9 + 10
- Keep Slide 12 unchanged

This keeps the full story but shortens delivery.
